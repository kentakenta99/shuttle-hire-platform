# デモ トークスクリプト — ホテル営業用
**対象:** ホテル総支配人・フロントマネージャー・宿泊部門責任者  
**所要時間:** 20分（質疑応答込み）  
**推奨デバイス:** ノートPC（ホテルスタッフ画面）＋スマホ（ゲスト画面）

---

## 事前準備（デモ5分前）

```
ノートPC:
  Tab 1 → https://shuttle.tokyomk.com/hotel/login    （ログイン前の状態）
  Tab 2 → https://shuttle.tokyomk.com/hotel/calendar （ログイン後 — 事前ログイン済み）
  Tab 3 → https://shuttle.tokyomk.com/admin          （TMK管理者 — 別ブラウザかシークレット）

スマホ:
  → /confirm/OKU-DEMO000005 を開いておく（TANAKA KENJI のQRチケット）
```

---

## INTRO — つかみ（1分）

**【言うこと】**
> 「今日お見せするのは、フロントスタッフが電話もFAXも使わずに、成田行きシャトルハイヤーを予約・管理できるシステムです。
> 操作は3ステップ、慣れれば1件30秒かかりません。
> まずゲストが実際に手にするものをご覧ください。」

*→ スマホを手に取る*

---

## シナリオ C — ゲスト体験（3分）

> ゲストが受け取るQRチケットから始める。「これが全ての起点です」

**【操作】**
スマホの `/confirm/OKU-DEMO000005` を相手に向ける

**【言うこと】**
> 「田中様というゲストの乗車チケットです。予約番号・出発日時・QRコードが表示されています。
> ゲストはこの画面をスマホに保存しておくだけ。当日ドライバーにこれを見せれば乗れます。」

💡 **フック:** スマホを渡してゲストとして触ってもらう

**【操作】**
「キャンセル」リンクをタップ → OTPメール入力画面を表示

**【言うこと】**
> 「キャンセルはゲスト自身がスマホで完結します。OTP（ワンタイムパスワード）をメールで受け取り入力するので、フロントへの電話が不要。深夜の問い合わせも来ません。」

---

## シナリオ A — ホテルスタッフ視点（8分）

### A-1: ログイン

**【操作】** Tab 1（ログイン画面）を表示

**【言うこと】**
> 「フロントスタッフはこの画面からログインします。ホテル共有アカウントなので、誰がログインしていても同じ画面が出ます。シフト引き継ぎも不要です。」

→ メールアドレス・パスワードを入力してログイン

---

### A-2: カレンダー（残席リアルタイム）

**【操作】** `/hotel/calendar` を表示

**【言うこと】**
> 「ログインするとこのカレンダーが表示されます。今日から1週間分の便が、残席数と一緒に並んでいます。
> 赤い枠が満席、緑が空きです。電話で『今日の9時は何席空いていますか』と聞かれても、この画面を見れば即答できます。」

💡 **フック:** 「ちなみにこれ、リアルタイムで更新されます」と言いながら Tab 2（別ウィンドウのカレンダー）を並べて表示

---

### A-3: 予約入力（ライブデモ）

**【操作】** 本日 11:30 便をクリック

**【言うこと】**
> 「では実際に予約を入れてみます。ゲストから『明日の成田行き、2名お願いしたい』と声をかけられた想定です。」

**【操作】** 「予約する」ボタンをクリック → フォームを表示

**【言うこと】**
> 「ゲスト名・人数・荷物数・フライト番号を入力します。フライト番号を入力すると…」

→ フライト番号「NH005」と入力

**【言うこと】**
> 「このように航空会社名とターミナルが自動で入ります。ANA便ならターミナル1、JALなら2、という判定が自動なので、ゲストに聞き直す必要がありません。」

💡 **フック:** 「AviationStackのAPIで飛行機の情報をリアルタイム取得しています」と一言添える

**【操作】** フォームを入力して「予約を確定する」

**【言うこと】**
> 「確定するとゲストのメールアドレスにQRチケットが自動送信されます。今見ていただいたスマホの画面と同じものが届きます。フロントからの作業はここまでで終わりです。」

---

### A-4: 予約一覧・キャンセル

**【操作】** `/hotel/bookings` を表示

**【言うこと】**
> 「予約一覧では、今日から先の予約がすべて確認できます。ゲストからキャンセルの連絡が来た場合は、ここから1クリックでキャンセルできます。」

---

## シナリオ B — TMK管理者視点（5分）

**【操作】** Tab 3（管理者画面 `/admin`）に切り替え

**【言うこと】**
> 「こちらは東京MK側の管理画面です。ホテル側には見えない裏側です。」

### B-1: ダッシュボード

**【言うこと】**
> 「今日・明日の便の予約状況がリアルタイムで集計されています。何台の車を準備すればよいか、一目でわかります。」

### B-2: ドライバーアサイン

**【操作】** 11:30 便の詳細を開く → 「ドライバーを選択」

**【言うこと】**
> 「予約が集まった便に、ここからドライバーをアサインします。社員番号で検索して選ぶだけです。アサインした情報はドライバー側の画面にも即座に反映されます。」

💡 **フック:** 「ドライバーがスマホで乗客名簿を確認できる仕組みも開発中です」と触れる

### B-3: CSV出力

**【操作】** 予約一覧 → CSVダウンロードボタン

**【言うこと】**
> 「月末の請求書作成はこのCSVをベースにできます。複数のホテルが使っていても、ホテル別に絞り込んで出力できます。」

>

## よくある質問 — 即答ガイド

| 質問 | 回答 |
|---|---|
| **費用はいくらですか？** | 「現在パイロット段階のため、費用の詳細は別途ご提案させてください。まず実際に使っていただいてご意見をいただく形を取っています」 |
| **決済はどうなっていますか？** | 「現在は月末請求まとめ払いです。オンライン決済は今後のロードマップに入っています」 |
| **GPSで車の位置はわかりますか？** | 「現バージョンには含まれていません。ロードマップにあり、ご要望いただければ優先度を上げます」 |
| **ドライバーへの通知は？** | 「現在はアサイン後にメールで通知しています。プッシュ通知対応のドライバーアプリは開発中です」 |
| **複数ホテルに展開できますか？** | 「DBは複数ホテル対応済みです。ホテルごとに完全に独立した管理画面と予約データになります」 |
| **スマホしか使えませんか？** | 「PCブラウザでも同じ画面が使えます。スマホ・タブレット・PC、すべて対応しています」 |
| **フライト遅延が起きたら？** | 「将来的にフライトAPIの有料プランを使えば、遅延情報をリアルタイムで取得して警告表示できる設計です」 |
| **セキュリティは大丈夫？** | 「ゲストのキャンセルはOTP認証必須、管理アクセスはSupabase Auth（MFA対応）で保護しています」 |
