System Overview

電話ゼロ・リアルタイム予約

ホテルのフロント・ベルデスクがシャトル便の空き枠をリアルタイムで確認し、 成田空港送りの予約を電話なしで完結。QRコードでチケットを発行し、 ドライバーが現場でスキャンして乗車確認まで行うエンドツーエンドシステムです。

📱

電話不要の即時予約

ホテルスタッフがブラウザから即日予約。空き枠・残席はリアルタイム更新。満席時は自動的に案内文を表示。

🎫

QRチケット発行

予約完了時にQRコードを発行。ゲストのスマートフォンで読み取れるデジタルチケット。乗車ナンバープレートも表示。

🛫

フライト情報自動取得

AviationStack APIと連携。乗務員画面にフライト出発時刻・遅延情報をリアルタイム表示。ターミナル停車順も自動提案。

🔐

3ロール権限設計

ホテルスタッフ／TMK管理者／ドライバー の3種類のアカウントを完全分離。各画面はロールに最適化されたUI。

📊

売上・稼働率ダッシュボード

スーパー管理者専用の統計画面。月次売上推移・便別稼働率・ホテル別予約ランキングを一覧表示。

🔒

セキュリティ監視

ログイン成功・失敗・不正アクセスを全件記録。不審IPの自動検出と管理者アラートで不正ログインを防止。

New in v2.1 — Guest Self-Booking

ゲストセルフ予約LP NEW

ホテルのQRチラシをスキャンするだけ。ゲストが自分で必要事項を入力し、 コンシェルジュに送信できるホテルブランドのランディングページ。 メールアドレス不要、アプリ不要、認証不要。

🎫

QRチラシ → ゲストLP → ホテル確認 → QRチケット

チラシのQRコードをスキャンすると、そのホテル専用のブランドページが開く。ゲストが自分で情報を入力し、コンシェルジュに「リクエスト」を送信。チラシ配りだけで予約チャネルが完結する。

New in v2.1 — Dynamic Pricing

動的価格・最安値確約システム NEW

同じ便に後から別のゲストが加わるほど、全員の料金が下がる仕組み。 いち早く予約したゲストも後から値引きの恩恵を受けられる。 キャンセルで値段が上がることは絶対にない。

Step 1

1名で予約 → ¥20,000

ゲストAが1名で予約。単価¥20,000が設定される。

Step 2

別ゲスト2名が同便に合流

ゲストBが2名で同じ便にリクエスト。同便の合計が3名に。

Step 3

全員の料金を再計算

3名ティア(¥10,000/名)が適用。AもBも全員が新価格に更新。

Step 4

🎉 自動通知

QRチケット・乗務員画面に「料金が安くなりました！」通知が表示。

価格変動の例（3名便の場合）

ゲストA ¥20,000 → ¥10,000 -¥10,000 自動割引
ゲストB (2名) ¥12,000 → ¥10,000 -¥2,000/名
合計: 3名 × ¥10,000 = ¥30,000

⚠️ 最安値確約ポリシー

キャンセルが発生しても、既存ゲストの料金は絶対に上がらない。

  • 価格は下がるか、据え置きのみ
  • キャンセル後の差額はTMKが負担
  • ゲストにとって最大リスクゼロの設計

Security

セキュリティ設計

不正ログイン検知・アクセス制御・データ保護を多層で実装。

🔐 認証・アクセス制御

  • 3ロール完全分離（hotel_staff / tmk_admin / driver）
  • Row Level Security（Supabase RLS）で全テーブル保護
  • 管理操作はサービスロールキー + 呼び出し元ロール検証
  • Refresh Token Rotation + 競合対策（getSession()）
  • IP別レートリミット（60分に3件）でスパム予約を防止

📋 認証イベントログ

  • ログイン成功・失敗・不正アクセスを全件記録
  • IPアドレス・UserAgentを記録（auth_eventsテーブル）
  • 1時間以内に3回以上失敗したIPを不審IPとして検出
  • 管理画面でリアルタイムアラート表示

🛡️ データ保護

  • APIキー・シークレットは環境変数のみ（コードにハードコード禁止）
  • SUPABASE_SERVICE_ROLE_KEY はサーバーサイドのみ参照
  • ゲスト確認URLは確認コードのみエンコード（フルURL不使用）
  • ログに個人情報・認証情報を出力しない

🏗️ インフラ

  • Vercel（US-East, iad1）上でホスト
  • Supabase PostgreSQL + 自動バックアップ
  • HTTPS強制（Vercel自動）
  • 環境変数はVercel Environment Variables管理

技術スタック

運用コストを最小化しつつスケール可能な構成。全てマネージドサービスで自社インフラ不要。

New in v2.2 — Operational Quality

v2.2 運用品質アップデート NEW

ホテルオークラ東京での本格稼働に向けて、メール通知の実装・セキュリティ強化・ スタッフUX改善を一括実施。

📧

メール通知 稼働開始

RESEND連携が完了し本番稼働中。予約確定・キャンセル時にゲストとホテルへQRコード付きメールを自動送信。

🛡️

満席エラー時は「空き枠一覧に戻る」リンク付きオレンジバナーを表示し、送信ボタンを自動無効化。予期しないサーバーエラー時の専用エラー画面も追加。

❓

ホテルスタッフFAQ

ナビバーに「FAQ」を追加。予約フロー・キャンセル・メール・料金・トラブル対応の5セクション・18問をアコーディオン形式で掲載。

✅

本番稼働確認済み
