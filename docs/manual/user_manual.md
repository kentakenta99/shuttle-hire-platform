## 1. システム概要

### 1-1. サービスの目的

MKシャトルハイヤー予約システムは、ホテルのフロント・ベルデスクが成田空港行きシャトルハイヤーの空き枠をリアルタイムで確認し、電話不要で予約を完結できるB2B予約システムです。

### 1-2. ご利用料金

| 種別 | 料金 |
|---|---|
| 1席あたり | ¥13,500（税込）※出発枠ごとに設定 |
| 最大定員 | 4席（満席時は通常ハイヤーにてご案内） |

### 1-3. ログインURL一覧

| 利用者 | URL |
|---|---|
| ホテルスタッフ | `https://shuttle-hire-platform.vercel.app/hotel/login` |
| TMK管理者 | `https://shuttle-hire-platform.vercel.app/admin/login` |
| 乗務員 | `https://shuttle-hire-platform.vercel.app/driver/login` |

### 1-4. 利用者ロール

| ロール | 役割 | アクセス先 |
|---|---|---|
| ホテルスタッフ | 予約の入力・照会 | `/hotel/` 配下 |
| TMK管理者 | 出発枠・予約の全体管理 | `/admin/` 配下 |
| 乗務員 | 担当便の乗客確認・乗車確認 | `/driver/` 配下 |
| スーパー管理者 | ユーザーCRUD・売上統計 | `/admin/superadmin/` 配下（権限付与が必要） |

## 2. ホテルスタッフ向けマニュアル

### 2-1. ログイン

#### 手順

1. ブラウザで `https://shuttle-hire-platform.vercel.app/hotel/login` を開く
2. **メールアドレス** と **パスワード** を入力する（東京エムケイより発行済みのホテル共有アカウント）
3. 「ログイン」ボタンをタップ → カレンダー画面に移動

#### 注意事項

- ログイン情報はフロント共有アカウントです。個人アカウントではありません
- 無操作 **60分** で自動ログアウトされます
- タブ・ブラウザを閉じると自動的にログアウトされます
- パスワードを忘れた場合は東京エムケイ配車センターにご連絡ください

---

### 2-2. 空き枠の確認（カレンダービュー）

ログイン後、本日以降の出発枠が日付ごとに一覧表示されます。

#### 画面の見方

```
┌────────────────────────────────────────┐
│  2026年6月20日（土）                    │
├────────────────────────────────────────┤
│  09:00発  💺💺　　　  残2席  [予約する] │
│  13:00発  💺💺💺💺  満席   ────────   │
│  16:00発  受付終了            ────────  │
└────────────────────────────────────────┘
```

| 表示 | 意味 |
|---|---|
| 残○席 / 💺（青） | 予約可能。「予約する」ボタンが押せます |
| 満席 | 予約不可。通常ハイヤーにてご案内ください |
| 受付終了 | 出発1時間前を過ぎて締切済 |
| 運休 | 東京エムケイ都合による運休 |

> **残席はリアルタイムで更新されます。** 他のホテルスタッフが同時に予約操作をしている場合、表示が変わることがあります。

---

### 2-3. 予約の入力

1. カレンダーから希望の便の「予約する」をタップ
2. 以下の情報を入力する

| 項目 | 入力内容 | 必須 |
|---|---|---|
| お客様のお名前 | 例：山田 太郎 / YAMADA, Taro | ✅ |
| 人数 | 1〜残席数の範囲で選択 | ✅ |
| フライト番号 | 例：NH832 / EK319 | ✅ |
| お荷物の数 | スーツケース等の大型荷物の個数 | ✅ |
| ご予約者名 | ベルボーイ・担当スタッフの名前（識別用） | 任意 |
| 備考 | 車椅子・英語対応希望など特記事項 | 任意 |

3. 「予約を確定する」ボタンをタップ

> **注意：** 「人数」は残席以上は選択できません。残席が足りない場合は、通常ハイヤーをご案内ください。

---

### 2-4. QRチケット（ゲストへのご案内）

予約が完了すると、**QRコードが画面に表示**されます。

#### QRチケットの内容

- **確認番号**（例：TMK-202606-0042）
- QRコード（ゲストのスマートフォンでスキャン）
- **乗車予定車両のナンバープレート**（例：品川 500 あ 1234）
- 出発日時・人数・フライト番号

#### ゲストへの案内方法

1. ゲストのスマートフォンでQRコードをスキャンしてもらう
2. スキャンすると、ゲストのスマートフォンにチケット画面が表示される
3. チケット画面には乗車車両のナンバープレートも表示されるため、当日はそのナンバーの車両に乗車する

> **印刷不要：** QRチケットはデジタルのみです。ゲストがスマートフォンでアクセスできない場合は、確認番号（TMK-XXXXXX-XXXX）を口頭でお伝えください。

---

### 2-5. 予約の照会・キャンセル

1. 左メニュー または ナビゲーションから **「予約一覧」** を選択
2. 確認番号・宿泊客名・フライト番号で検索可能
3. キャンセルする予約を選択 → 「キャンセルする」ボタンをタップ

> **注意：** キャンセルは **受付締切前（出発1時間前まで）** のみ可能です。締切後のキャンセルは東京エムケイ配車センターにお電話ください。

---

### 2-6. 満席時のご案内

予約できる枠が満席の場合は、以下のメッセージが表示されます。

```
この便は満席です。
通常ハイヤーでのご案内が可能です。
東京エムケイ 配車センターにご連絡ください。
```

## 3. TMK管理者向けマニュアル

ログインURL：`https://shuttle-hire-platform.vercel.app/admin/login`

---

### 3-1. ダッシュボード（トップ画面）

ログイン後すぐに本日の運行状況が表示されます。

| セクション | 内容 |
|---|---|
| 本日の便数 | 運行予定の出発枠数 |
| 本日の予約数 | 確定済み予約の件数・名数 |
| 今月の予約 | 月間累計 |
| 稼働率 | 今月の平均座席稼働率（%） |

---

### 3-2. 出発枠の管理

**ナビゲーション：** 「出発枠」タブを選択

#### 出発枠の新規作成

1. 「＋ 新規作成」ボタンをクリック
2. 以下を入力して保存

| 項目 | 内容 |
|---|---|
| 日付 | 運行日 |
| 出発時刻 | 任意の時刻（制限なし） |
| 定員 | 1〜（通常4席） |
| 車両種別 | スタンダードハイヤー / プレミアムハイヤー |
| 車両ナンバー | 担当車両のナンバープレート（例：品川 500 あ 1234）|
| 受付締切 | 出発何時間前に締め切るか（デフォルト1時間前） |
| 単価 | 1席あたりの料金（デフォルト¥13,500） |

> **車両ナンバーはゲストのQRチケットに表示されます。** 乗務員アサイン後に必ず入力してください。

#### 出発枠のステータス

| ステータス | 意味 | 変更方法 |
|---|---|---|
| `open` | 受付中・残席あり | 自動（残席あり時） |
| `full` | 満席 | 自動（残席0時） |
| `closed` | 受付終了 | 自動（締切時刻到達時） |
| `suspended` | 運休（TMK都合） | 手動・またはゼロ予約キャンセル確定時 |

---

### 3-3. 予約の管理

**ナビゲーション：** 「予約」タブを選択

- 日付・ホテル・ステータスでフィルタリング可能
- 管理者権限でのキャンセル・備考編集が可能
- 「CSVエクスポート」で請求書作成用データを出力

---

### 3-4. ドライバーアサイン

**ナビゲーション：** 出発枠詳細 → 「DR割当」ボタン

1. 出発枠の詳細画面を開く
2. 担当乗務員を選択（社員番号・氏名で表示）
3. 車両ナンバープレートを入力
4. 「保存」をクリック

> アサイン完了すると、乗務員の担当便一覧に当該便が表示されます。

---

### 3-5. ホテル管理

**ナビゲーション：** 「ホテル」タブ

- ホテルアカウントの一覧・詳細確認
- 月次利用実績の確認
- パスワードリセットはスーパー管理者が実施

---

### 3-6. 請求管理

**ナビゲーション：** 「請求」タブ

1. 対象月・対象ホテルを選択
2. 「レポート生成」をクリック
3. 内容を確認してPDF/CSVで出力

| 請求ステータス | 意味 |
|---|---|
| 未発行 | まだ請求書を発行していない |
| 発行済 | ホテルへ請求書を送付済み |
| 入金済 | 入金確認済み |

---

### 3-7. セキュリティ監視

**ナビゲーション：** 「🔒 セキュリティ」タブ

不正ログイン試行を監視する画面です。

#### 確認できる内容

| 項目 | 内容 |
|---|---|
| 認証ログ | ログイン成功・失敗の記録（IPアドレス・UserAgent付き） |
| 不審IPアラート | 1時間以内に3回以上ログイン失敗したIPを自動検出 |
| 最近のイベント | 直近のログイン試行一覧 |

> 不審なアクセスを発見した場合は、スーパー管理者にご連絡ください。

---

### 3-8. ゼロ予約キャンセルのフロー

出発時刻になっても予約が0件の場合：

```
乗務員が「予約ゼロ・15分待機」を確認
    ↓（15分後）
乗務員が「配車係に通知する」ボタンをタップ
    ↓
管理ダッシュボードに「⚠️ キャンセル待ち」アラート表示
    ↓
配車担当が確認 → 「運休確定」ボタン
    ↓
shuttle_slots.status が「suspended」に変更
乗務員に「運休確定」の通知が届く
```

### 4-2. 本日の担当便一覧

ログイン後すぐに本日の担当便が表示されます。

```
┌────────────────────────────────────────┐
│  2026年6月20日（土）                    │
├────────────────────────────────────────┤
```

### 4-3. 乗客名簿画面（便詳細）

担当便をタップすると乗客名簿が表示されます。

#### 画面の見方

| 表示項目 | 説明 |
|---|---|
| 乗客人数（大きい数字） | 本便の全搭乗者数 |
| 荷物数（黄色い数字） | 全荷物の総数 |
| フライト番号 | 各乗客のフライト |

| 予約ステータス | 「予約OK」 |

### 4-4. ターミナル停車順（自動表示）

フライト情報が取得できた乗客がいる場合、**ターミナル停車順が自動的に表示**されます。

```
┌────────────────────────────────────────┐
│  🛫 ターミナル停車順（自動）            │
│                                        │
│  第1停車  T2  → 第2停車  T1            │
│  3名        →   1名                    │
│                                        │
│  最早出発便の順に自動ソートしています   │
└────────────────────────────────────────┘
```

> **停車順は最早出発便のターミナルを先に訪問する**よう自動計算されています。フライト情報が取得できない場合は表示されません。

#### ターミナルバッジの色

| バッジ | 意味 |
|---|---|
| **T2** （濃紺） | 第2ターミナル（ANA・JALなど）|
| **T1** （濃紺） | 第1ターミナル（その他航空会社）|

---

### 4-5. 乗車確認（搭乗確認）

#### QRスキャンによる確認

1. 乗客名の横にある「搭乗確認」ボタンをタップ
2. カメラが起動するのでゲストのQRコードをスキャン
3. スキャン成功 → 乗客のステータスが「乗車確認済み」に変わる

#### 確認番号による手動入力

QRスキャンができない場合：
1. 「搭乗確認」ボタンをタップ
2. 「確認番号を手入力」を選択
3. ゲストの確認番号（TMK-XXXXXX-XXXX）を入力

---

### 4-6. 出発前の確認

全乗客の搭乗確認が完了すると、**「空港到着確認」ボタン**が表示されます。

1. ホテルを出発する前に全員の搭乗確認を完了させる
2. 空港到着後に「空港到着確認」ボタンをタップ
3. 便のステータスが完了に変わる

---

### 4-7. フライト情報について

乗客名簿には各乗客のフライト情報が表示されます。

| 表示 | 意味 |
|---|---|
| `NH832 ✓定刻` | フライトは定刻通り |
| `NH832 ⚠+20分遅延` | 20分の遅延が発生 |
| `NH832 （情報なし）` | フライト情報を取得できなかった |

> フライト情報はAviationStack APIからリアルタイムに取得しています。フライト出発の数時間前から取得可能になります。

---

### 4-8. ゼロ予約通知

出発時刻になっても予約がゼロの場合：

1. 「予約ゼロ・15分待機」の画面が表示される
2. 15分カウントダウンが表示される
3. 15分後に「配車係に通知する」ボタンをタップ
4. 管理者が確認・判断する（「運休確定」または「出発してください」）

> 15分以内にギリギリ予約が入った場合、タイマーはキャンセルされて通常フローに戻ります。

## 5. スーパー管理者向けマニュアル

アクセスURL：`https://shuttle-hire-platform.vercel.app/admin/superadmin`

> **アクセス権限：** `is_super_admin` フラグが付与されたTMK管理者アカウントのみアクセス可能です。ナビゲーションに「👑 スーパー管理者」が表示されます。

---

### 5-1. 売上ダッシュボード

**「ダッシュボード」タブ**で以下が確認できます。

| KPI | 内容 |
|---|---|
| 今月の予約数 | 当月の確定予約件数（キャンセル除く） |
| 今月の推定売上 | 予約人数 × 1席単価の合計 |
| 今月の平均稼働率 | 全出発枠の座席稼働率の平均 |
| アクティブユーザー | H（ホテル）/ DR（乗務員）/ A（管理者）の有効アカウント数 |

#### 月次推移チャート

過去6ヶ月の予約数推移をバーチャートで表示。月ごとの予約数・名数・推定売上も表形式で確認できます。

#### ホテル別予約ランキング

当月の予約件数が多い順にホテルをランキング表示。進捗バーで比率を視覚化。

#### 便別稼働率

当月の全出発枠の稼働率を一覧表示。

| 色 | 意味 |
|---|---|
| 青 | 稼働率60%未満 |
| オレンジ | 稼働率60〜89% |
| 赤 | 稼働率90%以上 |

---

### 5-2. ユーザー管理

**「ユーザー管理」タブ**で以下の3種類のアカウントを管理できます。

2. 「＋ ホテル追加」ボタンをクリック
3. 以下を入力して「作成する」をクリック

| 項目 | 必須 |
|---|---|
| ホテル名 | ✅ |
| ログインメール（認証用） | ✅ |
| 初期パスワード | ✅ |
| お迎え住所 | ✅ |
| 担当者名 | 任意 |
| 請求先メール | 任意（空白時はログインメールを使用） |

#### 管理者アカウントの追加

1. 「管理者」タブを選択
2. 「＋ 管理者追加」ボタンをクリック
3. 以下を入力して「作成する」をクリック

| 項目 | 必須 |
|---|---|
| 氏名 | ✅ |
| ログインメール | ✅ |
| 初期パスワード | ✅ |
| スーパー管理者権限を付与 | 任意チェックボックス |

#### アカウントの有効化/無効化

各アカウント行の右端にある「有効化」/「無効化」ボタンをクリックします。

> **乗務員アカウントについて：** 乗務員マスタは **Dispatch OS（DOS）で管理**されています。本システムでは乗務員の一覧確認のみ可能で、新規作成・編集はDOSで行ってください。

## 6. トラブルシューティング

### Q: ログインできない

**確認事項：**
1. URLが正しいか確認する（`https://shuttle-hire-platform.vercel.app/hotel/login`）
2. Caps Lockがオフになっているか確認
3. メールアドレスのスペルを確認
4. ブラウザのキャッシュをクリアして再試行

**解決しない場合：** 東京エムケイ配車センターに「アカウントのパスワードリセット依頼」としてご連絡ください。

---

### Q: 予約ボタンが押せない

**原因と対処：**

| 状態 | 対処 |
|---|---|
| 満席 | 通常ハイヤーをご案内ください |
| 受付終了 | 締切時刻を過ぎています。別の便を選ぶか配車センターへ |
| 「操作中」表示 | 他のスタッフが同時操作中です。数秒待って再試行 |

---

### Q: QRコードがスキャンできない

1. 画面の明るさを最大にする
2. カメラのレンズを清潔に保つ
3. スキャン距離を10〜30cm程度に調整する
4. QRスキャンができない場合は **確認番号（TMK-XXXXXX-XXXX）を口頭でお伝え**ください

---

### Q: フライト情報が表示されない

フライト情報はAviationStack APIから取得しています。以下の場合は表示されません：

- 出発日の数時間前より前の段階（まだAPIにデータがない）
- フライト番号の入力が正しくない（例：NH832 → 先頭はアルファベット2文字+数字）
- APIの一時的な障害

フライト情報がなくても乗車確認は通常通り行えます。

---

### Q: 「申し訳ありません。ちょうど満席になりました」と表示される

予約操作中に他のスタッフが同じ席を先に確保した場合に表示されます。カレンダービューに戻り、別の便・残席をご確認ください。

---

### Q: 画面が正しく表示されない

1. ブラウザのキャッシュをクリアする（Ctrl+Shift+R または Cmd+Shift+R）
2. ブラウザを最新版に更新する（Chrome推奨）
3. シークレットモードで再試行する
