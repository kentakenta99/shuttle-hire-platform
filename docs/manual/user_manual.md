## 1. システム概要

### 1-1. サービスの目的

MKシャトルハイヤー予約システムは、ホテルのフロント・ベルデスクが成田空港行きシャトルハイヤーの空き枠をリアルタイムで確認し、電話不要で予約を完結できるB2B予約システムです。

### 1-2. ご利用料金

| 種別 | 料金 |
|---|---|
| 1席あたり | ¥13,500（税込） |
| 最大定員 | 4席（満席時は通常ハイヤーにてご案内） |

### 1-3. 運行時間帯

出発時刻は **11:00〜15:00** の範囲内で設定されます。

### 1-4. ログインURL一覧

| 利用者 | URL |
|---|---|
| ホテルスタッフ | `shuttle.tokyomk.com/hotel/login` |
| TMK管理者 | `shuttle.tokyomk.com/admin/login` |
| 乗務員 | `shuttle.tokyomk.com/driver/login` |

## 2. ホテルスタッフ向けマニュアル

### 2-1. ログイン

#### 手順

1. ブラウザで `shuttle.tokyomk.com/hotel/login` を開く
2. **メールアドレス** と **パスワード** を入力する（東京エムケイより発行済みのホテル共有アカウント）
3. 「ログイン」ボタンをタップ → カレンダー画面に移動

#### 注意事項

- ログイン情報はフロント共有アカウントです。個人アカウントではありません
- 無操作 **60分** で自動ログアウトされます
- タブ・ブラウザを閉じると自動的にログアウトされます
- パスワードを忘れた場合は東京エムケイ配車センターにご連絡ください

---

### 2-2. 空き枠の確認（カレンダービュー）

ログイン後、本日の出発枠が一覧表示されます。

#### 画面の見方

```
┌────────────────────────────────────────┐
│  2026年6月20日（土）                    │
├────────────────────────────────────────┤
│  11:30発  残3席  [予約する]            │
│  13:00発  満席   ────────              │
│  14:30発  残1席  [予約する]            │
└────────────────────────────────────────┘
```

| 表示 | 意味 |
|---|---|
| 残○席 | 予約可能。「予約する」ボタンが押せます |
| 満席 | 予約不可。通常ハイヤーにてご案内ください |
| 締切済 | 受付終了（出発1時間前に自動締切） |
| 運休 | 東京エムケイ都合による運休 |

> **残席はリアルタイムで更新されます。** 他のホテルスタッフが同時に予約操作をしている場合、表示が変わることがあります。

---

### 2-3. 予約の入力

1. カレンダーから希望の便の「予約する」をタップ
2. 以下の情報を入力する

| 項目 | 入力内容 | 必須 |
|---|---|---|
| お客様のお名前 | 例：山田 太郎 / YAMADA, Taro | ✅ |
| 人数 | 例：2名 | ✅ |
| フライト番号 | 例：NH832 | ✅ |
| お荷物の個数 | 例：3個（スーツケース） | ✅ |
| 備考 | ベビーカーあり等 | 任意 |
| 予約スタッフ名 | ベルボーイ・担当者名 | 任意 |

3. 「予約を確定する」をタップ

#### 満席になった場合

```
この便は満席です
通常ハイヤーでのご案内が可能です。
東京エムケイにご連絡ください。
（特別割引料金にてご対応いたします）

配車センター：03-XXXX-XXXX
```

別の便を選ぶか、上記に記載の配車センターにお電話ください。

### 2-5. 予約の確認・キャンセル

#### 予約履歴の確認

1. 画面下部またはメニューから「予約履歴」を選択
2. 当日・翌日・過去30日分の予約が一覧表示される
3. 確認番号・お客様名・フライト番号で検索できる

#### キャンセルの手順

1. 予約履歴から対象の予約をタップ
2. 「キャンセルする」をタップ
3. 確認ダイアログで「はい」を選択

> **キャンセルポリシー：**  
> テスト期間中は無料キャンセル可。出発1時間前（締切時刻）を過ぎるとキャンセル操作が行えなくなります。その場合は東京エムケイ配車センターにご連絡ください。

---

### 2-6. 自動ログアウトについて

- 無操作 **60分** で自動ログアウトされます
- 残り **5分** になるとポップアップが表示されます：
  ```
  まもなくログアウトします。続行しますか？
  [はい、続行する]   [ログアウト]
  ```
- 「はい、続行する」を押すとタイマーがリセットされます

## 3. TMK管理者向けマニュアル

### 3-1. ログイン

1. `shuttle.tokyomk.com/admin/login` を開く
2. 管理者メールアドレスとパスワードを入力
3. 「ログイン」→ 管理ダッシュボードへ

---

### 3-2. ダッシュボード

ログイン後すぐに確認できる情報：

- **本日・翌日の出発枠** と予約埋まり率
- **直近の新規予約・キャンセル** 通知ログ
- **月次売上サマリー**（当月実績）
- **⚠️ キャンセル待ちアラート**（ゼロ予約便がある場合）

---

### 3-3. 出発枠の管理

#### A. 出発枠の作成（単発）

1. 左メニュー「出発枠」→「新規作成」
2. 以下を入力する：

| 項目 | 内容 |
|---|---|
| 日付 | 例：2026年6月20日 |
| 出発時刻 | 11:00〜15:00の間で設定 |
| 定員 | 例：4席 |
| 受付締切 | 出発1時間前が推奨 |
| 1席単価 | ¥13,500（変更可） |
| 備考 | 任意（社内メモ用） |

3. 「作成する」をタップ

#### B. 出発枠の一括作成（繰り返し設定）

1. 「出発枠一覧」→「繰り返し作成」
2. 開始日・終了日・曜日（例：月〜金）・時刻・定員を入力
3. 「祝日をスキップする」オプションあり
4. 「作成する」で指定期間分を一括登録

#### C. 出発枠のステータス変更

| ステータス | 意味 | 変更可能者 |
|---|---|---|
| open（受付中） | ホテル予約受付中 | 管理者 |
| full（満席） | 自動遷移（残席0で） | 自動 |
| closed（締切済） | 受付締切後に自動遷移 | 自動 |
| suspended（運休） | TMK都合による運休 | 管理者 |

> **注意：** `suspended`（運休）と `cancelled`（キャンセル）は別概念です。便全体の運休は `suspended`、個別乗客のキャンセルは各予約の `cancelled` で管理されます。

---

### 3-4. 予約管理

#### 全予約の確認

「予約一覧」から日付・ホテル・ステータスでフィルタして表示できます。

表示項目：
- 確認番号 / お客様名 / ホテル名 / 出発時刻 / 人数 / 荷物 / フライト番号 / ステータス

#### CSV出力

「CSV出力」ボタンから当月または指定期間の予約データをダウンロードできます。

#### 管理者によるキャンセル

1. 「予約一覧」から対象予約をクリック
2. 「管理者キャンセル」ボタン
3. キャンセル理由を入力して確定

---

### 3-5. ドライバーアサイン

1. 「出発枠一覧」から対象の便を選択
2. 「ドライバーアサイン」タブ
3. ドライバー名（社員コード）と車両番号を入力
4. 「アサインを確定」

アサイン完了後、対象ドライバーのアプリ（`/driver/`）に自動的に該当便が表示されます。

> **アサイン後の変更は管理者のみ可能です。**

---

### 3-6. 便ごとの乗車リスト

1. 出発枠詳細画面の「乗車リスト」タブ
2. 乗客一覧が表示される

No. | 氏名           | 人数 | 荷物 | フライト  | ホテル
  1 | 山田 太郎      |   3  |   3  | NH832    | オークラ

---

### 3-7. ゼロ予約便の対応（運休処理）

乗務員から「予約ゼロ・通知」が送られてきた場合、ダッシュボードに表示されます。

```
⚠️ キャンセル待ち
2026年6月20日 13:00発 — 予約0件
[乗務員より通知 15分前]

[運休確定]  [出発してください（キャンセル取消）]
```

- **「運休確定」を押した場合：** 便を `suspended` に変更。予約がある場合はホテルへメール自動送信
- **「出発してください」を押した場合：** 乗務員端末に「出発」通知が届き、通常運行

---

### 3-8. 月次請求レポート

1. 「請求管理」から対象月・対象ホテルを選択
2. 「レポート生成」

出力内容：日付 / 便 / 乗客名 / 人数 / 単価 / 小計 / 合計

請求ステータス：

| ステータス | 意味 |
|---|---|
| 未発行 | 請求書未送付 |
| 発行済 | 月末締め・翌月5日請求後 |
| 入金済 | 入金確認後に更新 |

### 4-4. QRコードのスキャン

ホテルスタッフが予約完了時にQRコードをゲストに提示します。

1. 詳細画面の「QRスキャン」ボタンをタップ
2. カメラが起動するのでQRコードを読み取る

### 4-5. ステータスの更新

```

```

### 4-6. ゼロ予約通知（キャンセル待ち）

出発時刻になっても予約が **0件** の場合：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  ⚠️ 予約件数：0件
  
  15分お待ちください
  [カウントダウン: 14:45]
  
  ※ ギリギリ予約が入る場合があります
  
  [配車係に通知する]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**手順：**

1. 15分のカウントダウンを待つ
2. その間に予約が入ればタイマーが消えて通常フローへ戻る
3. 15分経過してもゼロのまま → 「配車係に通知する」をタップ
4. TMK管理側で「運休確定」または「出発してください」の応答を待つ
5. 応答が届くまで待機

> **自動で配車係に通知されるわけではありません。** 必ず乗務員がボタンをタップして通知してください。

## 5. トラブルシューティング

### Q. ログインできない

- 入力したメールアドレス・パスワードをもう一度確認してください
- 大文字・小文字にご注意ください
- 解決しない場合は東京エムケイ配車センターにお電話ください

### Q. 予約ボタンが押せない

- **締切時刻を過ぎた場合：** 受付終了となります。配車センターに直接お問い合わせください
- **満席の場合：** 別の便をお選びいただくか、通常ハイヤーにてご案内ください

### Q. 残席が急に0になった

別のホテルスタッフが同時に予約を完了したためです。リアルタイムで残席が更新されます。別の便をお選びください。

### Q. QRコードが読み取れない

- タブレットのカメラへのアクセス許可が必要です。ブラウザの設定でカメラを「許可」にしてください
- QRコード画面の明るさを上げてみてください
- 電子署名方式（タブレット画面上にゲスト直接署名）に切り替えてください

### Q. 自動ログアウトされてしまった

ログイン画面からもう一度ログインしてください。予約データは保存されていますので、「予約履歴」からご確認いただけます。

### Q. 乗車リストが表示されない（乗務員）

ドライバーアサインが完了していない可能性があります。配車センターにご確認ください。
